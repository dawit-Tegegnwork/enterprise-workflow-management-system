# Deploy to Render (free tier)

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/dawit-Tegegnwork/enterprise-workflow-management-system)

## One-click deploy

1. Click **Deploy to Render** above.
2. After deploy, open `https://<your-service>.onrender.com/docs`.
3. Log in with demo accounts from the README (e.g. `staff@demo.local` / `Demo123!`).

## Environment variables

| Variable | Notes |
|----------|-------|
| `WORKFLOW_DATABASE_URL` | Defaults to ephemeral SQLite on free tier |
| `WORKFLOW_SECRET_KEY` | Auto-generated by Render when using `render.yaml` |

## Health check

```bash
curl https://<your-service>.onrender.com/health
```

Expected: `{"status":"ok"}`

## Demo workflow

```bash
TOKEN=$(curl -s -X POST https://<your-service>.onrender.com/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"staff@demo.local","password":"Demo123!"}' | jq -r .access_token)

curl -s -X POST https://<your-service>.onrender.com/api/v1/requests \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"title":"Cloud demo request","description":"Synthetic","department":"Ops"}'
```
